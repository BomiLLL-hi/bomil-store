# 6. АУТЕНТИФИКАЦИЯ И РОЛИ

Регистрация и авторизация **обязательны.**

**Причины:**
- Только авторизованный пользователь может оставить отзыв
- История заказов доступна только авторизованному пользователю
- История чата сохраняется только для авторизованных пользователей

**Способы входа:**
- Email + пароль
- Возможно: Google OAuth (опционально)

**Роли пользователей:**
- `user` — обычный покупатель
- `operator` — модератор (доступ к CMS: заказы, чат, очередь)
- `admin` — полный доступ к CMS включая настройки и управление операторами

**Доступ к CMS:**
- Обычный пользователь (`user`) не имеет доступа к `/admin`
- При попытке зайти на `/admin` без нужной роли → редирект на главную

**Что доступно без регистрации:**
- Просмотр каталога
- Поиск
- Добавление в корзину

**Что требует регистрации:**
- Оформление заказа (checkout)
- История заказов
- Отзывы
- История чата

**В профиле пользователя:**
- Имя (никнейм на сайте)
- Email
- Roblox username (привязывается при первом заказе или в профиле)
- История заказов
- Кнопка "История заказов"

# 7. СИСТЕМА ТОВАРОВ

Примерно 250 товаров. Все товары добавляются вручную через CMS.

**Каждый товар поддерживает:**
- `title`
- `slug`
- `aliases[]`
- `description`
- `images[]` — массив URL из Supabase Storage
- `category`
- `type` (knife / gun / pet / other)
- `stock_status` (in_stock / sold_out)
- `current_price` — цена в RUB (базовая валюта)
- `old_price` (опционально) — в RUB
- `hidden_status`
- `reviews`
- `included_items[]` (для сетов)
- `is_set` (boolean)
- `is_best_of_all_time` (boolean — добавляется вручную админом)

**Хранение изображений:**
- Все изображения товаров хранятся в **Supabase Storage** (bucket: `product-images`)
- Загрузка через CMS
- Используется Next.js Image для оптимизации

**Цены и валюты:**
- Все цены хранятся в базе в **рублях (RUB)**
- Конвертация в USD происходит на фронтенде
- Курс USD/RUB берётся из настроек CMS (вручную устанавливает админ)
- Например: админ ставит курс 90 ₽ = 1 $, и все цены автоматически пересчитываются

# 16. ROBLOX USERNAME WIDGET

На странице checkout есть шаг выбора Roblox-аккаунта.

**Как это работает:**
1. Пользователь вводит свой никнейм в поиск
2. Используется Roblox API (`https://users.roblox.com/v1/users/search?keyword=...`)
3. Появляются результаты: аватар + никнейм (как в интерфейсе Roblox)
4. Пользователь нажимает на свой аккаунт и подтверждает выбор
5. Выбранный username записывается в заказ

**Важно:**
- Пользователь может получить заказ ТОЛЬКО на тот username, который указан в заказе
- Это защищает от мошенничества

**Примечание:** Roblox API публичный, не требует ключей для поиска пользователей.

# 17. ПЛАТЁЖНАЯ СИСТЕМА

> ⚠️ Конкретный провайдер будет выбран позже. Архитектура должна быть готова к подключению любого из них.

**Рекомендуемые варианты (для РФ, ИП):**
- ЮKassa (Яндекс) — наиболее популярный, хорошая документация
- Robokassa — альтернатива
- CryptoCloud — если нужна крипта или международные платежи

**Архитектура платежей:**

```
Пользователь нажимает "Оплатить"
        ↓
Создаётся заказ со статусом "pending"
        ↓
Пользователь перенаправляется на страницу платёжного провайдера
        ↓
[Успешная оплата] → статус меняется на "paid" → страница подтверждения заказа
[Неуспешная оплата] → пользователь возвращается на главную страницу, корзина сохраняется
[Зависшая оплата] → статус остаётся "pending", через 30 минут автоматически отменяется, пользователю приходит письмо с предложением повторить оплату
        ↓
Провайдер отправляет webhook на сервер
        ↓
Сервер обновляет статус заказа
```

**Возврат средств:**
- Возврат возможен ТОЛЬКО если покупатель не получил заказ
- Возврат НЕ возможен, если заказ уже доставлен
- Возврат осуществляется через Live Chat (покупатель пишет, модератор обрабатывает вручную)
- В CMS: кнопка "Refund" появляется только если статус `paid` или `delivered`

**Статусы заказа:**

- `paid` — оплачен, ожидает выдачи
- `delivered` — выдан покупателю
- `refunded` — возврат средств
- `cancelled` — отменён (например, оплата зависла и истекло время)

**История заказов:**
- Доступна только авторизованным пользователям
- В профиле: кнопка "История заказов"
- Пользователь видит все свои заказы с номерами и статусами

# 19. ПРОЦЕСС ВЫДАЧИ ЗАКАЗА (МОДЕРАТОРОМ)

Выдача товаров происходит вручную модератором (не ботом).

**Процесс:**
1. Покупатель пишет номер заказа в Live Chat
2. Бот автоматически находит заказ и показывает позицию в очереди и примерное время ожидания
3. Модератор видит очередь в CMS (раздел "Live Chat / Queue")
4. Модератор добавляет покупателя в друзья в Roblox по его username
5. Они заходят вместе на сервер
6. Модератор передаёт товар
7. Модератор нажимает "Отметить как выдано" в CMS → статус заказа меняется на `delivered`

**Как модератор видит очередь:**

В CMS, раздел **"Очередь выдачи":**
- Таблица со всеми заказами со статусом `paid`
- Колонки: номер заказа, товары, Roblox username, время оплаты, позиция в очереди
- Кнопка "Взять в работу" → статус меняется на "in progress"
- Кнопка "Отметить как выдано" → статус меняется на `delivered`
- Если несколько операторов — каждый видит, кто взял заказ в работу

**Формула ожидания:**
- 1 позиция в очереди = ~2 минуты

# 23. LIVE CHAT

Floating кнопка в правом нижнем углу. Popup-style, expandable.

**Технические требования:**
- Realtime messaging (через Supabase Realtime)
- Anti-spam (cooldown между сообщениями)
- Auto replies (бот)
- Unread notifications (счётчик на иконке)
- Звуковые уведомления

**История чата:**
- Сохраняется только для авторизованных пользователей
- Хранится в Supabase
- Хранится до тех пор, пока заказ не помечен как `delivered`, + ещё 7 дней после этого

**Множество операторов:**
- Несколько модераторов могут работать одновременно
- В CMS видно, какой оператор ведёт какой чат

# 35. SEO

- Метатеги (title, description) для каждой страницы

- Sitemap (`/sitemap.xml`)
- `robots.txt`
- Canonical URLs
- Быстрая загрузка (влияет на SEO)

# 41. ФАЗЫ РАЗРАБОТКИ

Разработка делится на 6 фаз. Каждая фаза — самостоятельный блок.

---

## ФАЗА 1 — Каталог

- Структура базы данных (товары, категории)
- Страница каталога (product grid)
- Category bar + фильтры
- Страница товара (product page)
- Поиск (Fuse.js)
- Recently viewed
- Система сетов
- SEO (метатеги, sitemap)

---

## ФАЗА 2 — Аутентификация + Корзина + Checkout

- Регистрация / вход (email + пароль)
- Профиль пользователя
- Cart (корзина через localStorage)
- Checkout (форма + Roblox Widget)
- Юридические страницы (Terms, Refund Policy, Privacy Policy)

---

## ФАЗА 3 — Платёжная система + Заказы

- Интеграция платёжного провайдера (подключается позже)
- Webhook обработка
- Создание заказа
- Страница подтверждения заказа
- История заказов (в профиле)
- Email-уведомления
- Обработка зависших платежей (автоотмена через 30 минут)

---

## ФАЗА 4 — Live Chat

- Floating chat widget (frontend)
- Realtime messaging (Supabase Realtime)
- Бот (auto replies, flow "Получить заказ" / "Задать вопрос")
- Anti-spam / cooldown
- Сохранение истории чата
- Уведомления (счётчик, звук)

---

## ФАЗА 5 — CMS / Админка

- Dashboard
- Products Management
- Order Management
- Queue Management (очередь выдачи)
- Reviews Management
- Live Chat Management (тикеты + ответы)
- FAQ Management
- Settings (операторы, настройки сайта)

---

## ФАЗА 6 — Отзывы + Главная страница

- Система отзывов (написание, модерация, отображение)
- Блок Latest Reviews / Latest Orders (carousel)
- FAQ на главной странице
- Trust elements
- Финальная полировка UI/UX
- Тестирование
- Deploy на Vercel
